# 游戏规则

## 概述

纸牌消除游戏：主牌区的牌可以与顶部开放顶牌匹配消除，消除所有主牌区牌即为胜利。

## 牌局构成

- **主牌区**：多张牌按布局排列，有覆盖关系
- **开放顶牌**：1~3 张面朝上的牌（开局时选择难度决定数量）
- **底牌堆**：面朝下的储备牌
- **废牌堆**：被替换掉的旧顶牌
- **奖励牌**：金色特殊牌，消除时额外补充底牌（由配置 `rewardCardsPerBonus` 控制，默认 3 张）

## 操作

### 匹配

双击主牌区翻开的牌，如果该牌与任一开放顶牌点数差等于 `matchRankDiff`（配置，默认 1），则匹配成功：

- 主牌区该牌飞向顶牌区（动画时长 `matchFlyDuration`）
- 匹配的顶牌被替换为新牌
- 下层被压住的牌自动翻开
- 所有卡槽重新计算高亮状态

### 抽牌

点击底牌堆，从储备牌中抽一张到开放顶牌区：

- 如果开放顶牌数已达上限（由 `visibleTopCardCountMax` 配置），最旧的顶牌移入废牌堆
- 新牌添加到开放顶牌末尾，带缩放动画

### 回收

底牌堆耗尽且废牌堆非空时，出现回收按钮：

- 废牌堆洗牌后合并到底牌堆
- 清空操作历史（不可撤销回收操作）

### 撤销

撤销最近一步操作：

- **撤销匹配**：牌从顶牌区飞回原卡槽，被翻开的牌翻回去
- **撤销抽牌**：抽到的牌退回底牌堆，旧顶牌从废牌堆恢复

### 拖放

可以拖动主牌区翻开的牌到顶牌区：

- 拖到顶牌区附近且可匹配 → 执行匹配
- 拖到顶牌区附近但不可匹配 → 显示"无法匹配"
- 拖到其他位置 → 弹回原位（时长 `bounceBackDuration`）

## 难度

开局选择翻开底牌数：

| 难度 | 开放顶牌数 | 对应配置 |
|------|-----------|---------|
| 困难 | `visibleTopCardCountMin`（默认 1） | 容错最低 |
| 普通 | 2 | 匹配机会适中 |
| 简单 | `visibleTopCardCountMax`（默认 3） | 更多匹配机会 |

## 奖励牌

部分布局含奖励牌槽位。消除奖励牌时，从奖励牌下方的底牌堆飞出 `rewardCardsPerBonus` 张（配置，默认 3）底牌到储备堆，带依次延迟动画（`rewardFlyDelay`）。

## 胜利条件

所有主牌区卡槽为空 → 显示 "YOU WIN!"（由 `strings.json` 配置）。

## 关卡切换

点击 "NEXT" 按钮循环切换内置布局，切换时重置牌局。

## 动画参数

所有动画时长均可通过 `game_config.json` 配置：

| 配置键 | 默认值（秒） | 说明 |
|--------|------------|------|
| `matchFlyDuration` | 0.3 | 匹配时牌飞向顶牌区 |
| `flipDuration` | 0.2 | 翻牌动画 |
| `bounceBackDuration` | 0.25 | 拖放失败弹回 |
| `drawDelayDuration` | 0.3 | 抽牌后延迟解锁 |
| `undoFlyDuration` | 0.3 | 撤销匹配时牌飞回 |
| `topCardScaleUpDuration` | 0.2 | 新顶牌缩放出现 |
| `topCardScaleUpFrom` | 0.8 | 新顶牌初始缩放值 |
| `rewardFlyDelay` | 0.15 | 奖励牌飞出的间隔延迟 |
| `rewardFlyDuration` | 0.25 | 奖励牌单张飞行时长 |

动画期间输入锁定，防止重复操作。
